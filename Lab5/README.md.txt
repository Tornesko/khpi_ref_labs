# Опис прийнятих рішень щодо рефакторингу

У цьому рефакторингу було використано патерн проектування "Стратегія" для зменшення залежності класу Product від конкретних аспектів функціональності. Було створено базовий клас ProductSearchStrategy та його конкретні підкласи, які відповідають різним стратегіям пошуку товарів (за ім'ям, за типом). Це дозволяє легко змінювати та додавати нові методи пошуку без зміни коду класу Product. Також був створений клас ProductManager для керування пошуком та відображенням товарів за обраною стратегією.

Цей підхід покращує структуру коду, зменшуючи його залежності та полегшуючи розширення функціональності.
